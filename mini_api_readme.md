# 看图猜成语游戏流程分析

## 初始化阶段
1. **音效配置获取**（soundConfig）
   - 获取游戏所有音效文件，包括按钮点击、答对、答错等效果
   - 所有音效都有对应的名称和URL地址

2. **统计配置获取**（aldConfig）
   - 获取游戏中各种行为的统计ID配置
   - 用于追踪用户游戏行为和数据分析

3. **其他基础配置**
   - hotWord：检查热门词功能状态（当前isBeta=false）
   - guideList：获取新手引导（当前为空）
   - previewLevel：获取关卡预览（当前为空）

## 游戏主流程
1. **获取关卡信息**（gameLevel）
   - 服务器返回当前关卡（从第23关开始）
   - 每个关卡包含多个题目（每题对应一个成语）
   - 题目包含图片URL、答案和提示信息
   - 题目类型主要有：
     - type=1：纯看图猜成语
     - type=2：带有成语解释的猜成语

2. **用户登录**（login）
   - 获取用户openId和session信息
   - 绑定用户ID（862647）

3. **答题循环**
   - 用户查看图片，猜测成语
   - 提交答案（pass接口）
   - 参数包含：关卡level、用时passTime和验证码code
   - 服务器返回结果，包含：
     - 是否正确
     - 是否升级（upgrade字段）
     - 本关所有成语列表
     - 进度提示（如"再过2关即可升级土坯房"）

4. **进度系统**
   - 游戏设有房屋升级机制
   - 随着关卡推进，玩家可从"土坯房"升级到"驴车"等
   - 每完成特定关卡会触发upgrade=true

5. **数据上报**（uploadFormId）
   - 每完成一关后上传表单ID
   - 用于数据统计或后续消息推送

## 技术特点
1. 所有请求均使用GET方法
2. URL中包含sign参数用于请求验证
3. 服务器响应格式统一：包含code、msg、time和data字段
4. 用户答题时间被记录并上传到服务器
5. 每关通过后会获取下一关信息，形成连续游戏体验
